### 3.2 新增方法

**文件**: `src/environment_manager.py`

#### install_skill_dependencies()

```python
async def install_skill_dependencies(
    self,
    agent_name: str,
    skill_path: Path,
    skill_name: str
) -> Tuple[bool, str, List[str]]:
    """
    为 Skill 安装依赖

    检测 Skill 目录下的 scripts/requirements.txt 文件，
    如果存在且尚未安装，则自动安装依赖包。

    Args:
        agent_name: Agent名称
        skill_path: Skill 目录路径（Skill 的根目录）
        skill_name: Skill名称

    Returns:
        (success, message, installed_packages) - 成功标志、消息、已安装的包列表
    """
```

**实现逻辑**:
1. 检测 `{skill_path}/scripts/requirements.txt` 是否存在
2. 读取并解析依赖列表（跳过注释和空行）
3. 检查 `installed_dependencies[skill_name]` 是否已安装
4. 调用 `install_packages()` 安装依赖
5. 更新 `installed_dependencies` 记录

#### check_skill_dependencies_installed()

### 4.1 修改 execute_script()

在 `execute_script()` 方法中，获取环境后立即安装依赖：

```python
async def execute_script(self, ...):
    # ... 创建执行记录 ...

    async with semaphore:
        try:
            # 获取或创建环境
            env = await self.environment_manager.get_or_create_environment(agent_name)

            # 新增：安装 Skill 依赖
            if skill_base_path:
                skill_path = Path(skill_base_path).parent
                success, message, packages = await self.environment_manager.install_skill_dependencies(
                    agent_name=agent_name,
                    skill_path=skill_path,
                    skill_name=skill_name
                )
                if not success:
                    print(f"[EXEC] 警告: 依赖安装失败 - {message}")
                elif packages:
                    print(f"[EXEC] 已安装依赖: {packages}")

            # 准备工作目录...
            # 执行脚本...
```

## 十、注意事项

1. **依赖安装时机**: 依赖在首次执行 Skill 时安装，不是在环境创建时
2. **依赖缓存**: 已安装的依赖会记录在 `installed_dependencies` 中，避免重复安装
3. **错误处理**: 依赖安装失败不会阻塞执行，会打印警告并继续
4. **文件路径**: 脚本中的文件路径使用 `./input/` 作为输入目录
5. **输出格式**: 所有脚本输出必须是 JSON 格式，包含 `status` 字段
