#### 3.2.2 文件上传函数

```typescript
const uploadPendingFiles = useCallback(async (files: PendingFile[]): Promise<FileAttachment[]> => {
  if (files.length === 0) return [];

  const uploadedFiles: FileAttachment[] = [];
  const failedFiles: string[] = [];

  setIsUploading(true);
  setUploadProgress(0);

  for (let i = 0; i < files.length; i++) {
    const pf = files[i];
    setUploadingFileName(pf.name);
    setUploadProgress(Math.round((i / files.length) * 100));

    try {
      const uploadedFile = await uploadFile(agentName, pf.file);
      uploadedFiles.push({
        id: uploadedFile.id,
        name: uploadedFile.filename,
        size: uploadedFile.size,
        type: uploadedFile.mimeType
      });
    } catch (e) {
      failedFiles.push(pf.name);
    }
  }

  setUploadProgress(100);
  setIsUploading(false);

  return uploadedFiles;
}, [agentName, locale]);
```

#### 3.2.3 handleSend 中的文件处理

```typescript
const handleSend = useCallback(async () => {
  // 1. 先上传新的待发送文件
  let newUploadedFiles: FileAttachment[] = [];
  if (currentPendingFiles.length > 0) {
    newUploadedFiles = await uploadPendingFiles(currentPendingFiles);
  }

  // 2. 合并已有的文件上下文和新上传的文件
  const allFileIds = [...fileContext.file_ids, ...newUploadedFiles.map(f => f.id)];
  const allFileInfos = [...fileContext.file_infos, ...newUploadedFiles];

  // 3. 更新文件上下文（文件在对话中持续有效）
  if (newUploadedFiles.length > 0) {
    setFileContext({
      file_ids: allFileIds,
      file_infos: allFileInfos
    });
  }

  // 4. 发送请求时携带 file_ids
  const res = await fetch(requestUrl, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      message: userContent,
      history: historyMessages,
      file_ids: allFileIds  // 文件 ID 列表
    }),
  });
}, [/* ... */]);
```

**skill_loaded 事件**:
```typescript
if (data.type === 'skill_loaded') {
  const skillName = data.skill_name;
  const success = data.success;
  // 更新状态为 completed 或 failed
  streamingSkillStatesRef.current = streamingSkillStatesRef.current.map(s =>
```

**tool_call 事件 (execute_skill)**:

```typescript
      ? { ...s, status: 'executing', message: '正在执行...' }

  );
```

**tool_result 事件 (execute_skill)**:
```typescript
if (toolName === 'execute_skill') {
```

## 四、API 接口

### 4.1 文件上传 API

**端点**: `POST /api/agents/{name}/files`

**请求**: `FormData` with `file` field

**响应**:
```json
{
  "success": true,
  "file": {
    "file_id": "uuid",
    "filename": "report.pdf",
    "file_size": 2457600,
    "mime_type": "application/pdf",
    "uploaded_at": "2026-03-12T12:00:00Z"
  }
}
```

### 4.2 流式对话 API

**端点**: `POST /stream/agents/{name}/chat`

**请求**:
```json
{
  "message": "提取这个PDF的文本",
  "history": [...],
  "file_ids": ["uuid1", "uuid2"]
}
```

**响应**: SSE 流
```
data: {"type": "skill_loading", "skill_name": "AB-pdf"}
data: {"type": "skill_loaded", "skill_name": "AB-pdf", "success": true}
data: {"type": "tool_call", "name": "execute_skill", "args": {...}}
data: {"type": "tool_result", "name": "execute_skill", "result": "..."}
data: {"type": "content", "content": "文本内容..."}
data: {"type": "metrics", "first_token_latency": 500, ...}
```

## 六、注意事项

### 6.1 流式输出兼容性

- 所有修改都保持了对现有流式输出功能的兼容
- `flushSync` 继续用于打字机效果
- 新的状态更新使用 React 批处理

### 6.2 文件上下文持久性

- 文件上下文在整个对话期间持续有效
- 用户可以随时清除文件上下文
- 每次发送消息时，新上传的文件会合并到现有上下文

### 6.3 Skill 状态同步

- Skill 状态通过 SSE 事件实时更新
- 使用 `useRef` 存储流式状态，避免频繁重渲染
- 状态转换: loading → executing → completed/failed
