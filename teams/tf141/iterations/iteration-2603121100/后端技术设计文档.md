# 后端技术设计文档

## 1. 概述

### 1.1 背景

用户反馈问题：勾选的是 "AB-DOCX creation, editing, and analysis" 这个skill，但智能体执行时报错 "AB-docx" 这个skill不存在。

### 1.2 根因分析

1. **名称来源不一致**:
   - 前端展示可能使用SKILL.md的标题（如 "AB-DOCX creation, editing, and analysis"）
   - 后端注册时使用frontmatter中的name（如 "AB-docx"）
   - Agent配置保存的skills列表可能混合使用两种名称

2. **大小写敏感**:
   - 注册表中的key是 "ab-docx"（小写）
   - 查询时使用 "AB-docx"（混合大小写）导致匹配失败

3. **缺乏容错机制**:
   - `skill_tool.py` 只进行精确匹配，没有模糊匹配逻辑
   - 错误提示不够友好，未列出可用的skill ID

### 1.3 设计目标

- AC-2.1: 所有builtin skills的名称统一为小写连字符格式
- AC-2.4: 即使LLM使用了大小写错误的名称也能正确匹配
- AC-2.5: 技能加载失败时，错误信息中列出所有可用的Skill ID

def _normalize_skill_name(self, name: str) -> str:
    """
    规范化Skill名称为小写连字符格式

    规则:
        1. 统一转为小写
        2. 空格替换为连字符
        3. 多个连续连字符合并为一个
        4. 移除首尾连字符
    """
    if not name:
        return name

    normalized = name.lower()
    normalized = re.sub(r'\s+', '-', normalized)
    normalized = re.sub(r'-+', '-', normalized)
    normalized = normalized.strip('-')

    return normalized
```

def _format_available_skills(self) -> str:
    """
    格式化可用的Skill列表，用于错误提示
    """
    if not self.enabled_skills:
        return "No skills are currently enabled for this agent."

    lines = ["Available skills for this agent:"]
    for skill_name in self.enabled_skills:
        skill = self.skill_registry.get_skill(skill_name)
        if skill:
            desc = skill.description[:80] + "..." if len(skill.description) > 80 else skill.description
            lines.append(f"  - {skill_name}: {desc}")
        else:
            lines.append(f"  - {skill_name}")
    return "\n".join(lines)
```

## 3. 数据流

### 3.1 Skill加载流程

```
用户选择Skill (前端)
        ↓
保存到Agent配置 (可能使用原名称)
        ↓
LLM调用 load_skill (可能使用不同大小写)
        ↓
SkillTool._match_skill_name (模糊匹配)
        ↓
返回匹配后的名称或错误提示
```

### 3.2 名称规范化时机

| 时机 | 操作 |
|------|------|
| 扫描builtin skills | 使用规范化名称注册 |
| 加载skills_index.json | 保留原名称，但模糊匹配时规范化 |
| 查询skill | 模糊匹配时规范化查询名称 |

## 7. 附录

### 7.1 名称规范化规则

```
原始名称 -> 规范化名称
AB-docx -> ab-docx
AB-DOCX -> ab-docx
AB DOCX -> ab-docx
AB--DOCX -> ab-docx
AB-DOCX creation... -> ab-docx-creation
```

### 7.2 模糊匹配算法

1. 精确匹配 - O(1)
2. 规范化精确匹配 - O(n)
3. 前缀匹配 - O(n)
4. 包含匹配 - O(n)

总体复杂度: O(n)，n为enabled_skills数量

### 7.3 兼容性说明

- 向后兼容: 支持旧的名称格式
- 不影响现有配置: 配置文件中仍可使用原名称
- 渐进式迁移: 可逐步更新为规范化名称
