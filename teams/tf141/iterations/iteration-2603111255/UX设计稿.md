### 1.2 用户旅程状态图

```mermaid
stateDiagram-v2
    [*] --> 空闲状态: 进入对话页面

    空闲状态 --> 文件选择中: 点击上传按钮
    空闲状态 --> 拖拽悬停: 拖拽文件进入

    文件选择中 --> 文件预览: 选择有效文件
    文件选择中 --> 空闲状态: 取消选择

    拖拽悬停 --> 文件预览: 释放文件
    拖拽悬停 --> 空闲状态: 拖拽离开

    文件预览 --> 文件预览: 继续添加文件
    文件预览 --> 空闲状态: 删除所有文件

    文件预览 --> 上传中: 点击发送
    空闲状态 --> 对话请求中: 点击发送(无文件)

    上传中 --> 对话请求中: 上传成功
    上传中 --> 上传失败: 上传失败

    上传失败 --> 文件预览: 重试
    上传失败 --> 空闲状态: 取消

    对话请求中 --> 空闲状态: 完成
```

## 10. 设计决策记录

### 10.1 为什么选择回形针图标？

- 回形针是业界通用的"附件"符号
- 用户认知成本低
- 与邮件、聊天软件的附件按钮一致

### 10.2 为什么文件卡片悬停显示删除按钮？

- 默认隐藏可以保持界面简洁
- 悬停显示符合用户预期（类似 macOS Dock）
- 避免误操作（需要先悬停再点击）

### 10.3 为什么选择先上传再发送？

- 后端需要 file_id 来关联文件
- 上传失败时可以立即反馈，不影响对话
- 避免在对话过程中等待上传

### 10.4 为什么支持拖拽上传？

- 提升高级用户的效率
- 从文件管理器直接拖入比点击选择更快
- 符合现代 Web 应用的交互习惯
