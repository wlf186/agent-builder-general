## 1. 需求背景

### 1.1 问题描述
有客户反馈CoinGecko这个预置MCP服务在系统主页测试时显示"连接失败"，且在智能体中配置后无法被正常调用并返回结果。

**具体表现**：
- 系统主页MCP服务测试时，CoinGecko显示"连接失败"
- 用户在调试对话中输入"BTC的最新价格"，智能体好像感知不到有这个MCP
- 其他MCP服务（calculator、cold-jokes）测试正常

### 1.2 CTO诊断结果（2026-03-13）

**重要发现**：CoinGecko MCP服务在当前环境下**连接完全正常**！

| 诊断层级 | 状态 | 延迟 | 详情 |
|---------|-----|------|------|
| CONFIG | ✓ PASS | - | 配置验证通过 |
| DNS | ✓ PASS | 23ms | mcp.api.coingecko.com → 172.66.172.219 |
| NETWORK | ✓ PASS | 10ms | TCP连接成功 |
| TLS | ✓ PASS | 1563ms | TLSv1.3 |
| MCP | ✓ PASS | 3220ms | **50个工具可用** |

**结论**：服务实际HEALTHY，问题根因是**环境不一致**

### 1.3 根因分析

| 可能原因 | 可能性 | 说明 |
|---------|--------|------|
| 后端未使用虚拟环境启动 | ⭐⭐⭐ | 系统Python缺少mcp库，导致连接失败 |
| 偶发网络问题 | ⭐⭐ | 防火墙/代理/速率限制（现已恢复） |
| 状态显示错误 | ⭐⭐ | 前端测试端点与实际状态不一致 |
| 工具描述问题 | ⭐ | LLM无法识别工具名称 |

### 1.4 用户痛点分析

| 痛点层级 | 具体描述 | 解决方案 |
|---------|---------|----------|
| **显性痛点** | 测试显示"连接失败" | 确保环境一致性 |
| **隐性痛点** | 错误信息不明确 | 添加环境检查提示 |
| **深层痛点** | LLM不选择调用工具 | 优化工具描述 |

## 3. 功能需求（已更新）

### 3.1 需求一：后端启动环境检查

**优先级**: P0（必须实现）

**功能描述**：
在后端启动时检查MCP库依赖，如果缺失则给出明确提示。

**验收标准**：
- [ ] 启动时检测`mcp`库是否可用
- [ ] 缺失时显示明确的警告："MCP库未安装，远程MCP服务将不可用"
- [ ] 提供修复指引（如："请运行: pip install mcp"）

**用户故事**：
> 作为运维人员，我想在启动时就知道MCP功能是否可用，避免用户报告连接失败

**技术实现**：
```python
# backend.py 启动时添加
try:
    import mcp
    print(f"✓ MCP 库已加载 (版本 {mcp.__version__})")
except ImportError:
    print("✗ 警告: MCP 库未安装，远程 MCP 服务将不可用")
    print("  请运行: pip install mcp")
```

### 3.2 需求二：前端测试状态一致性

**优先级**: P0（必须实现）

**功能描述**：
确保前端MCP服务测试显示与后端实际状态一致。

**验收标准**：
- [ ] `/api/mcp-services/{name}/test`端点正确返回服务状态
- [ ] CoinGecko测试显示为"连接成功"
- [ ] 错误时提供具体原因（而非笼统的"连接失败"）

**技术考虑**：
- 检查`backend.py`中测试端点的实现
- 确保使用正确的Python环境（.venv）

### 3.3 需求三：集成诊断结果显示

**优先级**: P1（高优先级）

**功能描述**：
将后端诊断功能（`src/mcp_diagnostic.py`）集成到前端UI。

**验收标准**：
- [ ] 前端MCP服务对话框中显示"诊断连接"按钮
- [ ] 点击后展示分层诊断结果（DNS→Network→TLS→MCP）
- [ ] 显示修复建议

**技术考虑**：
- 后端诊断API已存在：`/api/mcp-services/{name}/diagnose`
- 前端组件已存在：`MCPDiagnosticResult.tsx`
- 需要确认API端点是否已正确暴露

### 3.4 需求四：CoinGecko工具描述优化

**优先级**: P2（中优先级）

**功能描述**：
优化CoinGecko MCP工具的描述，提高LLM识别率。

**验收标准**：
- [ ] 工具描述包含"加密货币"、"BTC"、"价格"等关键词
- [ ] 提供调用示例

**CTO诊断发现的工具**：
- get_coins_markets - 获取市场价格
- get_coins_by_id - 获取单个币种详情
- get_coin_market_chart - 获取市场图表

**验收标准**：
- [ ] 服务列表中显示连接状态指示器（绿色/黄色/红色）
- [ ] 鼠标悬停显示简要状态信息
- [ ] 点击可查看详细诊断

## 7. 数据模型

### 7.1 诊断报告（已存在）

```typescript
interface MCPDiagnosticReport {
  service_name: string;
  overall_status: "healthy" | "degraded" | "down";
  timestamp: string;
  layers: DiagnosticLayer[];
  recommendation: string;
}

interface DiagnosticLayer {
  layer: string;
  status: "pass" | "fail" | "skip";
  message: string;
  latency_ms?: number | null;
  details?: Record<string, any> | null;
}
```

### 7.2 服务配置增强

```typescript
interface MCPServiceConfig {
  // 现有字段...
  name: string;
  url: string;
  auth_type: "none" | "bearer" | "apikey";
  auth_value?: string;

  // 新增字段
  docs_url?: string;          // 服务文档链接
  setup_guide?: string;       // 配置指引
  health_check_url?: string;  // 健康检查端点
}
```
