## 4. 问题报告

### P0-001: 后端MCP库未安装

| 属性 | 值 |
|------|-----|
| 问题ID | P0-001 |
| 标题 | 后端服务运行时缺少MCP Python库 |
| 优先级 | P0 |
| 严重性 | 高 |
| 状态 | 待修复 |

#### 问题描述
后端服务启动时未使用虚拟环境（`.venv`），系统Python中未安装`mcp`库，导致远程SSE模式的MCP服务（如CoinGecko）无法连接。

#### 复现步骤
1. 确认后端服务未使用虚拟环境启动
2. 调用`POST /api/mcp-services/coingecko/diagnose`
3. 观察MCP层返回"fail"状态，消息为"MCP库未安装"

#### 实际结果
```json
{
  "layer": "mcp",
  "status": "fail",
  "message": "MCP库未安装"
}
```

#### 预期结果
```json
{
  "layer": "mcp",
  "status": "pass",
  "message": "MCP连接成功，获取到 50 个工具",
  "details": {
    "tool_count": 50
  }
}
```

#### 根本原因
- 后端启动时直接使用系统Python（`python backend.py`）
- 虚拟环境`.venv`中已安装`mcp 1.26.0`
- 系统`/usr/bin/python3`中没有`mcp`库

#### 修复方案
方案1（推荐）: 使用启动脚本
```bash
./start_backend.sh
```

方案2: 手动使用虚拟环境
```bash
cd /work/agent-builder-general
source .venv/bin/activate
python backend.py
```

方案3: 安装MCP库到系统Python
```bash
pip install mcp
```
