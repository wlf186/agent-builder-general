## 2. MCP服务诊断流程

### 2.1 诊断分层架构

```
用户点击"诊断连接"
        │
        ▼
┌─────────────────────────────────────┐
│ 1. 配置验证 (Config Validation)      │
│ - 检查URL格式                         │
│ - 检查认证配置                         │
│ - 验证必要参数                         │
└─────────────────────────────────────┘
        │ 配置无效 → 返回错误
        │ 配置有效
        ▼
┌─────────────────────────────────────┐
│ 2. DNS解析 (DNS Resolution)          │
│ - 解析域名到IP地址                    │
│ - 测量解析延迟                        │
└─────────────────────────────────────┘
        │ DNS失败 → 返回网络错误
        │ DNS成功
        ▼
┌─────────────────────────────────────┐
│ 3. 网络连接 (Network Connectivity)   │
│ - TCP连接测试                         │
│ - 测量连接延迟                        │
│ - 检测端口开放                        │
└─────────────────────────────────────┘
        │ TCP失败 → 返回网络错误
        │ TCP成功
        ▼
┌─────────────────────────────────────┐
│ 4. TLS/SSL握手 (TLS Handshake)       │
│ - HTTPS证书验证                       │
│ - TLS版本检测                         │
│ - 加密套件检测                        │
└─────────────────────────────────────┘
        │ TLS失败 → 返回证书错误
        │ TLS成功 (或HTTP)
        ▼
┌─────────────────────────────────────┐
│ 5. MCP协议 (MCP Protocol)            │
│ - SSE连接建立                         │
│ - 工具列表获取                        │
│ - 协议版本兼容性                      │
└─────────────────────────────────────┘
        │
        ▼
    生成诊断报告
```

| 属性 | 值 |
|------|-----|
| 服务名称 | coingecko |
| URL | https://mcp.api.coingecko.com/sse |

### 3.2 可能的问题场景

```
场景A: 需要API Key
─────────────────────
问题: 连接时返回401/403
解决: 在UI中引导用户配置API Key

场景B: 网络限制
─────────────────────
问题: DNS解析失败或连接超时
解决: 提供代理配置选项

场景C: 协议不兼容
─────────────────────
问题: MCP握手失败
解决: 显示协议版本，建议升级

场景D: 工具描述问题
─────────────────────
问题: LLM不选择调用
解决: 优化工具描述，添加关键词
```

## 7. 错误处理策略

### 7.1 错误分类

| 错误类别 | HTTP状态 | 恢复策略 |
|---------|---------|---------|
| 配置错误 | 400 | 提示用户修正配置 |
| 认证失败 | 401/403 | 提示配置API Key |
| 网络超时 | 504 | 提示检查网络/代理 |
| 服务不可用 | 503 | 提示稍后重试 |
| 协议错误 | 500 | 记录日志，联系技术支持 |

### 7.2 重试策略

```python
# 诊断重试策略
- 配置验证: 不重试（用户错误）
- DNS解析: 重试1次
- TCP连接: 重试2次，递增延迟
- TLS握手: 重试1次
- MCP协议: 重试2次

# 工具调用重试策略（已存在）
- max_retries = 2
- 重试延迟: 1秒
- 超时时间: 60秒
```

## 8. 性能指标

### 8.1 诊断性能目标

| 指标 | 目标值 | 测量方法 |
|------|--------|---------|
| 配置验证 | < 10ms | 内存操作 |
| DNS解析 | < 1000ms | 网络请求 |
| TCP连接 | < 2000ms | 连接建立 |
| TLS握手 | < 3000ms | SSL握手 |
| MCP协议 | < 5000ms | SSE连接 |
| **总诊断时间** | < 10000ms | 端到端 |

### 8.2 监控指标

- 诊断成功率
- 各层失败率
- 平均诊断耗时
- 服务可用性

## 9. 安全考虑

### 9.1 认证信息保护

- API Key不在日志中显示
- 前端传输时不明文存储
- 使用环境变量存储敏感信息

### 9.2 外部服务隔离

- 诊断请求超时控制
- 限制并发诊断数量
- 防止DNS劫持
