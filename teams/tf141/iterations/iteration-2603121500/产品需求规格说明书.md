# 产品需求规格说明书 (PRD)
## Skill执行状态显示异常修复

**版本**: 1.0
**创建日期**: 2026-03-12
**负责人**: TF141 产品经理
**优先级**: P0 (高优先级 - 影响用户体验)

---

## 1. 问题描述

### 1.1 问题现象
用户在使用 skill-test-pdf 和 skill-test-doc 智能体时，"技能执行状态"区域同时显示"执行完成"和"执行失败"两种状态。

### 1.2 用户影响
- 用户无法准确判断技能是否执行成功
- 降低用户对系统的信任度
- 影响调试对话的功能体验

---

## 2. 问题场景还原

### 2.1 测试环境
- **测试文件**: `test/测试1.pdf`, `test/测试2.docx`
- **智能体**: skill-test-pdf, skill-test-doc
- **技能**: AB-pdf, AB-docx

### 2.2 重现步骤
1. 打开调试对话页面
2. 进入 skill-test-pdf 或 skill-test-doc 智能体
3. 点击上传按钮，选择 `test/测试1.pdf` 或 `test/测试2.docx`
4. 输入指令：`提取文件内容`
5. 观察左下角"技能执行状态"区域

### 2.3 预期行为
```
┌─────────────────────────────────────┐
│ 技能执行状态                         │
│ ✓ AB-pdf        执行完成            │
└─────────────────────────────────────┘
```

### 2.4 实际行为
```
┌─────────────────────────────────────┐
│ 技能执行状态                         │
│ ✓ AB-pdf        执行完成            │
│ ✗ AB-pdf        执行失败            │
└─────────────────────────────────────┘
```

## 3. 影响范围分析

### 3.1 受影响的技能
经检查 `skills/builtin/` 目录，以下技能均使用 `execute_skill` 工具调用，可能受影响：

| 技能名称 | 功能描述 | 受影响程度 |
|---------|---------|-----------|
| AB-pdf | PDF文件处理 | 🔴 高 |
| AB-docx | Word文档处理 | 🔴 高 |
| AB-pptx | PowerPoint处理 | 🟡 中 |
| AB-xlsx | Excel处理 | 🟡 中 |
| AB-mcp-builder | MCP服务器构建 | 🟡 中 |
| AB-skill-creator | 技能创建工具 | 🟡 中 |
| AB-webapp-testing | Web应用测试 | 🟡 中 |
| AB-web-artifacts-builder | Web构件构建 | 🟡 中 |

### 3.2 问题类型分类
- **特定skill问题**: ❌ 否
- **通用问题**: ✅ 是 - 所有使用 `execute_skill` 的技能都会受影响

### 3.3 触发条件
- 单次对话中调用多个技能时
- 某个技能执行结果包含 "error"/"Error"/"失败" 字符串时
- 技能执行流程中同时存在 `skill_loading` 和 `tool_call` 事件时

## 4. 技术根因

### 4.1 问题代码位置
**文件**: `frontend/src/components/AgentChat.tsx`
**行号**: 535-541, 571-576

### 4.2 错误逻辑
```typescript
// ❌ 错误：使用 status 而非 skillName 匹配
streamingSkillStatesRef.current = streamingSkillStatesRef.current.map(s =>
  s.status === 'executing'    // 匹配所有 executing 状态的技能
    ? { ...s, status: isFailed ? 'failed' : 'completed', ... }
    : s
);
```

### 4.3 正确逻辑应为
```typescript
// ✅ 正确：使用 skillName 精确匹配
const skillName = extractSkillName(toolResult);  // 从 tool_result 中提取 skill_name
streamingSkillStatesRef.current = streamingSkillStatesRef.current.map(s =>
  s.skillName === skillName    // 仅匹配对应技能
    ? { ...s, status: isFailed ? 'failed' : 'completed', ... }
    : s
);
```
