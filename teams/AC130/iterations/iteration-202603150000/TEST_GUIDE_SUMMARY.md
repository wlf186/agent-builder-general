## 1. 核心教训

### 1.1 选择器错误（最严重的问题）

**错误**：使用 `page.locator('textarea').first()` 选中了左侧"人设与提示词"编辑框

**正确**：使用 `page.locator('input[type="text"][placeholder]').first()` 选中右侧聊天输入框

**原因分析**：
- 页面左侧配置区使用 `<textarea>` 组件（人设编辑）
- 页面右侧聊天区使用 `<input type="text">` 组件（消息输入）
- `textarea.first()` 返回第一个 textarea，即左侧配置区

### 1.2 后端未重启

**问题**：修复代码后未重启后端服务，导致 API 仍返回旧错误

**解决方案**：每次代码修改后必须重启后端：
```bash
kill $(cat backend.pid)
python backend.py
```

### 1.3 正常回复的定义

**之前**：只验证"有响应"，未验证响应内容是否正常

**现在**：必须验证响应内容是有意义的回答：
- 用户发送"你好" → AI 回复"你好！很高兴见到你！😊"
- 而不是空字符串、错误提示、或乱码

## 2. 测试检查清单

### 测试前必查

- [ ] 后端服务已重启（加载最新代码）
- [ ] API 直接测试通过（curl 测试）
- [ ] 使用正确的选择器定位聊天输入框

### 测试中必查

- [ ] 消息输入到**右侧聊天区域**
- [ ] AI 回复内容**有意义**（不是空/错误）
- [ ] 流式输出**流畅**（打字机效果）

### 测试后必查

- [ ] 截图证据已保存
- [ ] 截图中 AI 回复内容清晰可见
