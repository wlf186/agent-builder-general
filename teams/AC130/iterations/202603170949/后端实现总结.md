### 3.2 knowledge_base_manager.py 修改

#### 3.2.1 修复文档列表解析逻辑

**问题**：原代码使用 `split("_", 1)` 无法正确解析 `doc_xxxxx_filename` 格式

**修复**：使用 `find("_", 4)` 定位第二个下划线
```python
# 找到第二个下划线的位置
underscore_pos = stem.find("_", 4)  # 从第4个字符开始找（跳过 "doc_"）
if underscore_pos > 0:
    doc_id = stem[:underscore_pos]
    original_filename = stem[underscore_pos + 1:] + file_path.suffix
```

## 4. 技术架构

### 4.1 数据模型

```python
# 核心模型
KnowledgeBase    # 知识库元数据
Document         # 文档元数据
Chunk            # 文档块
RetrievalResult  # 检索结果
RetrievalConfig  # 检索配置（top_k, score_threshold, prompt_template）
```

### 4.2 处理流程

```
文档上传
    ↓
文档解析（PDF/DOCX/TXT）
    ↓
智能分块（RecursiveCharacterTextSplitter）
    ↓
向量化（BAAI/bge-small-zh-v1.5）
    ↓
存储（ChromaDB + 文件系统）
    ↓
检索（Cosine Similarity）
```

### 4.3 AgentEngine 集成

```python
# 1. 初始化时加载知识库检索器
self._init_retrievers()

# 2. 创建 RAG 检索工具
self._rag_tools = self._create_rag_tools()

# 3. 在对话中使用
# - 自动注入知识库上下文
# - 或提供 rag_retrieve 工具给 LLM 调用
```

## 7. 使用示例

### 7.1 创建知识库

```python
from src.knowledge_base_manager import KnowledgeBaseManager

kb_manager = KnowledgeBaseManager(Path("data"))

kb = kb_manager.create_kb(
    name="产品文档",
    description="公司产品相关文档"
)
print(f"知识库创建成功: {kb.kb_id}")
```

### 7.2 上传文档

```python
from pathlib import Path

doc = kb_manager.add_document(
    kb_id=kb.kb_id,
    file_path=Path("product_manual.pdf"),
    filename="产品手册.pdf"
)
print(f"文档已处理: {doc.chunk_count} 个块")
```

### 7.3 检索知识库

```python
retriever = kb_manager.get_retriever(kb.kb_id)
results = retriever.search("产品价格", top_k=3)

for result in results:
    print(f"[{result.filename}] {result.content[:50]}... (score: {result.score:.3f})")
```

### 7.4 关联到智能体

```python
from src.models import AgentConfig

config = AgentConfig(
    name="客服助手",
    persona="你是产品客服助手...",
    knowledge_bases=[kb.kb_id],  # 关联知识库
    retrieval_config=RetrievalConfig(
        top_k=3,
        score_threshold=0.6
    )
)
```

## 8. 已知问题和限制

### 8.1 已修复

- ✅ **文档列表解析错误**：修复了 `list_documents` 方法中文件名解析逻辑
- ✅ **AgentConfig 字段缺失**：添加了 `knowledge_bases` 和 `retrieval_config` 字段支持

### 8.2 当前限制

1. **嵌入模型**：默认使用 `BAAI/bge-small-zh-v1.5`，暂不支持多模型
2. **文件大小限制**：单个文件最大 10MB
3. **并发处理**：文档处理是同步的，大文件可能阻塞
4. **混合检索**：HybridRetriever 已预留但未实现 BM25
