## 5. 问题列表

### BUG-001: PRD 与 API 响应字段名不一致

| 项目 | 内容 |
|------|------|
| **严重程度** | Minor |
| **重现步骤** | 1. 调用 GET /api/system/check-conda<br>2. 比较响应字段与 PRD |
| **预期行为** | PRD 中描述的字段名与 API 一致 |
| **实际行为** | API 返回 `available/path/version/error/message`，PRD 中描述为 `conda_available/conda_path/environment_type` |
| **截图** | 无 |
| **状态** | Open - 需要更新 PRD |

**说明**: 前端类型定义 (`CondaCheckResult`) 与后端 API 响应一致，PRD 需要更新以匹配实际实现。

### BUG-002: 缺少系统库导致 Playwright 无法运行

| 项目 | 内容 |
|------|------|
| **严重程度** | Environmental |
| **重现步骤** | 在 WSL 环境中运行 npx playwright test |
| **预期行为** | Playwright 可以正常运行测试 |
| **实际行为** | Chrome headless shell 报错: `error while loading shared libraries: libnspr4.so` |
| **截图** | 见测试日志 |
| **状态** | Blocked - 需要安装系统依赖或使用有头模式 |

**解决方案**:
```bash
# 安装缺失的库
sudo apt-get update
sudo apt-get install -y libnspr4 libnss3 libatk1.0-0 libatk-bridge2.0-0 libcups2 libdrm2 libxkbcommon0 libxcomposite1 libxdamage1 libxfixes3 libxrandr2 libgbm1 libasound2
```
