## 1. 需求概述

### 背景问题

用户在创建新智能体时遇到环境初始化失败错误：
```
环境初始化失败
创建环境时发生错误: [Errno 2] No such file or directory: 'conda'
```

根本原因：
1. `get_conda_path()` 找不到 conda 时返回 `"conda"` 字符串
2. 执行时抛出 `[Errno 2]` 错误
3. 用户系统可能未安装 conda/miniconda/anaconda

### 迭代目标

1. **友好错误提示** - 提供清晰易懂的错误信息和解决指引
2. **Conda 可用性检测** - 创建智能体前预检测 conda 状态
3. **降级方案支持** - 支持 system-python 环境模式（模型定义）

### 决策 1: get_conda_path() 返回值处理

| 项目 | 内容 |
|------|------|
| **背景** | 原函数在找不到 conda 时返回 `"conda"` 字符串，导致后续执行时报错 `[Errno 2]` |
| **选项** | A: 返回 `"conda"` 保持原样<br>B: 返回 `None` 让调用方处理<br>C: 抛出异常 |
| **决策** | 选择 B - 返回 `None` |
| **理由** | 调用方可以根据返回值判断 conda 是否可用，提供更友好的错误提示 |
| **影响** | `_run_conda_command()` 需要检查返回值并抛出友好错误 |
