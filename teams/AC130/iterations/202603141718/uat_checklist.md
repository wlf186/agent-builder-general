## 测试原则

### 必须遵守

1. **使用真实浏览器操作**
   - ✅ 使用 Playwright 进行 UI 自动化测试
   - ❌ 严禁使用 curl 等后台命令替代

2. **从用户视角测试**
   - 仅根据 PRD 理解功能，脱离代码实现细节
   - 关注用户体验和业务价值

3. **提供证据**
   - 每个测试用例必须包含截图
   - 问题的重现步骤必须清晰

### 测试覆盖

- [ ] 冒烟测试：核心功能是否可用
- [ ] 功能测试：所有功能点是否按 PRD 实现
- [ ] 边界测试：边界条件处理是否正确
- [ ] 异常测试：异常场景是否有合理处理
- [ ] 体验测试：用户操作是否流畅
