## 1. 背景与目标

### 1.1 业务背景

当前系统采用 Conda 虚拟环境作为智能体技能执行的隔离方案（详见 `iteration-2603130900` 调查报告）。当用户创建新智能体时，系统会自动为该智能体创建独立的 Conda 环境。

**用户反馈问题**：
用户在新建智能体后看到以下报错信息：
```
环境初始化失败
创建环境时发生错误: [Errno 2] No such file or directory: 'conda'
```

**根本原因分析**（由 Lead 诊断）：

1. `src/environment_manager.py` 中的 `get_conda_path()` 函数在找不到 conda 时返回 "conda" 字符串
2. 当 `asyncio.create_subprocess_exec` 执行 "conda" 时，如果 PATH 中没有 conda，就会抛出 `[Errno 2]` 错误
3. 用户系统可能：
   - 未安装 conda/miniconda/anaconda
   - conda 安装在非标准路径
   - PATH 环境变量未正确配置

**当前代码逻辑** (`environment_manager.py:22-39`):
```python
def get_conda_path() -> str:
    """获取conda可执行文件路径"""
    # 优先使用环境变量
    if os.environ.get("CONDA_EXE"):
        return os.environ["CONDA_EXE"]
    # 检查常见安装位置
    common_paths = [
        "/opt/miniconda3/bin/conda",
        "/opt/conda/bin/conda",
        os.path.expanduser("~/miniconda3/bin/conda"),
        os.path.expanduser("~/anaconda3/bin/conda"),
        "/usr/local/miniconda3/bin/conda",
    ]
    for path in common_paths:
        if os.path.isfile(path):
            return path
    # 回退到PATH中的conda
    return "conda"  # 这里是问题所在 - 未验证 conda 是否可用
```

**用户体验问题**：
- 错误信息技术化，普通用户难以理解
- 没有明确的解决路径指引
- 无替代方案，用户完全被阻塞

### 1.2 目标

本次迭代旨在解决 conda 环境初始化失败问题，提升用户体验：

1. **友好错误提示**：当 conda 不可用时，提供清晰易懂的错误信息和解决指引
2. **降级方案**：提供系统 Python 作为备选方案，允许智能体在没有 conda 的情况下正常创建
3. **前端优化**：优化错误展示，提供操作指引链接
4. **环境检测**：在创建智能体前预检测 conda 可用性，提前告知用户

### 1.3 成功指标

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| 错误信息可理解性 | 用户可在 5 秒内理解错误含义 | 用户测试反馈 |
| 问题解决率 | 80% 用户可通过指引自行解决 | 后台错误率统计 |
| 创建成功率提升 | 新智能体创建成功率 > 95% | 成功率监控 |
| 用户满意度 | NPS > 8 | 用户调研 |

#### F-001: Conda 可用性检测

**用户故事**：
```
作为系统用户
我希望在创建智能体前就知道系统是否支持 conda
以便提前了解可能的限制并做出决策
```

**验收标准**：
- [ ] Given 用户访问创建智能体页面 When 页面加载完成 Then 自动检测 conda 可用性
- [ ] Given conda 不可用 When 检测完成 Then 显示警告提示，但允许继续创建
- [ ] Given conda 可用 When 检测完成 Then 显示正常状态，不干扰用户

#### F-003: 系统 Python 降级方案

**用户故事**：
```
作为没有安装 conda 的用户
我希望可以使用系统 Python 创建智能体
以便快速体验系统核心功能
```

**验收标准**：
- [ ] Given 用户选择"系统 Python"模式 When 创建智能体 Then 使用系统 Python 创建轻量级环境
- [ ] Given 使用系统 Python When 执行不需要特殊依赖的技能 Then 正常执行
- [ ] Given 使用系统 Python When 执行需要特殊依赖的技能 Then 显示友好错误，说明需要 conda

**环境模式对比**：

| 模式 | 隔离级别 | 依赖管理 | 适用场景 |
|------|----------|----------|----------|
| `conda` | 完全隔离 | 独立环境 | 生产环境、多技能场景 |
| `system-python` | 部分隔离 | 用户虚拟环境 | 快速体验、开发测试 |

**技术实现**：
```python
class EnvironmentMode(str, Enum):
    CONDA = "conda"           # 完整 Conda 虚拟环境
    SYSTEM_PYTHON = "system"  # 系统Python + venv
    SKIP = "skip"             # 跳过环境创建
```

- 技术细节默认折叠，可展开查看

## 3. 非功能需求

### 3.1 性能要求

| 指标 | 要求 |
|------|------|
| conda 检测响应时间 | < 2 秒 |
| 错误信息加载时间 | < 1 秒 |

### 3.3 兼容性要求

| 平台 | 支持情况 |
|------|----------|
| Linux | 完全支持 |
| macOS | 完全支持 |
| Windows (WSL) | 完全支持 |

### 4.2 交互规范

1. **检测时机**：进入创建智能体页面时静默检测，不阻塞用户操作
2. **警告展示**：使用 Toast + 警告卡片组合，确保用户注意到但不被打断
3. **错误恢复**：提供明确的下一步操作，避免用户卡住
4. **降级提示**：明确告知降级模式的功能限制

## 5. 技术约束

### 5.1 技术栈

| 组件 | 技术 |
|------|------|
| 后端 | Python 3.10+, FastAPI |
| 前端 | Next.js 15, React 19, TypeScript |
| 状态管理 | React Hooks |

### 5.2 依赖项

- 无新增外部依赖
- 需要复用现有的 `EnvironmentManager`

### 5.3 限制条件

1. 系统 Python 模式不提供完整的依赖隔离
2. 某些技能（如 PDF 处理）在系统 Python 模式下可能无法正常工作
3. Windows 原生环境暂不支持

## 6. 风险评估
