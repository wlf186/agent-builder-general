## 1. 需求概述

在系统层面建立统一的知识库管理模块，并将 RAG（检索增强生成）抽象为一种可插拔的"工具"，由智能体根据任务需求自主触发。

---

## 2. 技术方案

### 2.1 技术选型

| 维度 | 选型 | 理由 |
|------|------|------|
| 向量数据库 | ChromaDB | 轻量级、易部署、与 LangChain 集成良好 |
| 文档解析 | PyPDFium2 + python-docx | 支持 PDF/DOCX，性能稳定 |
| Embedding | BAAI/bge-small-zh-v1.5 | 中文效果好，无需 API 费用 |
| 文本分块 | RecursiveCharacterTextSplitter | LangChain 推荐，支持中文 |

## 4. API 端点

### 4.1 知识库管理

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/knowledge-bases` | 列出所有知识库 |
| POST | `/api/knowledge-bases` | 创建知识库 |
| GET | `/api/knowledge-bases/{id}` | 获取知识库详情 |
| DELETE | `/api/knowledge-bases/{id}` | 删除知识库 |

### 4.2 文档管理
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/knowledge-bases/{id}/documents` | 列出文档 |
| POST | `/api/knowledge-bases/{id}/documents` | 上传文档 |
| DELETE | `/api/knowledge-bases/{id}/documents/{doc_id}` | 删除文档 |
| POST | `/api/knowledge-bases/{id}/search` | 检索知识库 |
| GET | `/api/knowledge-bases/{id}/stats` | 获取统计信息 |
