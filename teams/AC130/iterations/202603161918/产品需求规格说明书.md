Agent Builder 知识库（RAG）管理系统及动态挂载机制

## 文档版本
- 版本: v1.0
- 日期: 2026-03-16

## 1. 需求背景与目标

### 1.1 背景
目前智能体仅依赖 Prompt 和外部 API（MCP/Skills），缺乏对企业私有文档的深度理解能力。当用户询问公司内部制度、产品手册等私有信息时，智能体无法给出准确回答。

### 1.2 目标
在系统层面建立统一的知识库管理模块，并将 RAG（检索增强生成）抽象为一种可插拔的"工具"，由智能体根据任务需求自主触发。

### 1.3 技术调研结论

| 维度 | 调研结论 | 推荐方案 |
|------|----------|----------|
| **向量数据库** | ChromaDB 适合轻量级/原型场景，易部署，与 LangChain 集成良好 | ChromaDB（本地持久化） |
| **文档解析** | Unstructured 支持多种格式，LangChain 有官方集成 | Unstructured + PyPDF/Docx |
| **Embedding** | HuggingFace 本地模型无需 API 费用，效果稳定 | sentence-transformers/all-MiniLM-L6-v2 |
| **RAG 模式** | Agent-based 模式让 LLM 自主决定是否检索 | Agent Tool 方式 |

**参考资源**：
- [LangChain RAG 官方文档](https://docs.langchain.com/oss/python/langchain/rag)
- [RAG Best Practices (arXiv 2024)](https://arxiv.org/abs/2407.01219)
- [ChromaDB vs Milvus 对比](https://zilliz.com/blog/milvus-vs-chroma)

## 2. 功能模块设计

### 2.1 全局知识库管理（系统主页新增菜单）

**位置**：系统主页侧边栏，与"MCP 服务"、"技能货架"平级

**功能列表**：

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 创建知识库 | 名称 + **描述（Description）** | P0 |
| 编辑知识库 | 修改名称和描述 | P0 |
| 删除知识库 | 删除库及其所有文档和向量 | P0 |
| 上传文档 | 支持 PDF、DOCX 格式 | P0 |
| 文档列表 | 显示已上传文档及其处理状态 | P0 |
| 删除文档 | 从知识库中删除单个文档 | P1 |

**元数据结构**：
```python
class KnowledgeBase:
    id: str                    # UUID
    name: str                  # 知识库名称
    description: str           # 描述（用于 LLM 判断是否调用）
    created_at: datetime       # 创建时间
    updated_at: datetime       # 更新时间
    document_count: int        # 文档数量
```

### 2.2 智能体配置关联

**位置**：智能体详情页 → 工具配置区域 → 新增"RAG 知识库"分类

**功能**：
- 多选框列表展示所有可用知识库
- 只有被勾选的知识库，其描述信息才会进入该智能体的 `Tools Definition`
- 未勾选的库对智能体完全不可见（数据隔离）

**Agent 配置扩展**：
```python
class AgentConfig:
    # ... 现有字段 ...
    knowledge_bases: list[str]  # 挂载的知识库 ID 列表
```

### 2.3 动态触发逻辑（运行时）

**触发流程**：
```
用户问题 → LLM Planning → 判断是否需要检索
                         ↓
               是：调用 rag_retrieve 工具
                         ↓
               返回相关文档片段 + 生成回答
                         ↓
               在回答中标注引用来源
```

**Tool Schema 设计**：
```json
{
  "name": "rag_retrieve",
  "description": "从挂载的知识库中检索相关文档内容。仅在需要查询内部文档、公司制度、产品手册等信息时使用。",
  "parameters": {
    "type": "object",
    "properties": {
      "query": {
        "type": "string",
        "description": "检索查询语句"
      },
      "kb_ids": {
        "type": "array",
        "items": {"type": "string"},
        "description": "要检索的知识库 ID 列表（可选，默认检索所有挂载的库）"
      }
    },
    "required": ["query"]
  }
}
```

┌───────────────────┐   ┌─────────────────┐   ┌─────────────────────────┐
│ DocumentProcessor │   │  Embedder       │   │  KnowledgeBaseManager   │

### 3.2 数据处理流水线

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   文档上传   │ →  │   文本提取   │ →  │   分块处理   │ →  │   向量化存储 │
│  (PDF/DOCX)  │    │ (Unstructured)│   │ (Chunking)   │    │  (ChromaDB)  │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
```

**分块策略**：
- Chunk Size: 500 字符
- Chunk Overlap: 100 字符
- 保留元数据：文档名、页码、章节标题

### 3.3 RAG 检索流程

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│   用户查询   │ →  │   向量化     │ →  │   相似检索   │ →  │   结果重排   │
│              │    │  (Embedding) │    │ (Top-K=5)    │    │  (可选)      │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
                                                                   │
                                                                   ▼
                                                            ┌──────────────┐
                                                            │   返回上下文 │
                                                            │ + 来源标注   │
                                                            └──────────────┘
```

## 4. API 接口设计

### 4.1 知识库管理 API

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/knowledge-bases` | 获取所有知识库列表 |
| POST | `/api/knowledge-bases` | 创建新知识库 |
| GET | `/api/knowledge-bases/{id}` | 获取知识库详情 |
| PUT | `/api/knowledge-bases/{id}` | 更新知识库信息 |
| DELETE | `/api/knowledge-bases/{id}` | 删除知识库 |

### 4.2 文档管理 API

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/knowledge-bases/{id}/documents` | 获取知识库文档列表 |
| POST | `/api/knowledge-bases/{id}/documents` | 上传文档到知识库 |
| DELETE | `/api/knowledge-bases/{id}/documents/{doc_id}` | 删除文档 |
| GET | `/api/knowledge-bases/{id}/documents/{doc_id}/status` | 获取文档处理状态 |

### 4.3 RAG 检索 API（内部）

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/api/knowledge-bases/retrieve` | 执行 RAG 检索 |

## 5. Prompt 注入设计

当智能体挂载了知识库时，系统需自动在 System Prompt 中加入：

```
## 知识库检索能力

你可以使用 `rag_retrieve` 工具从挂载的知识库中检索内部文档内容。

**可用知识库**：
{{#each knowledge_bases}}
- **{{name}}**: {{description}}
{{/each}}

**使用指南**：
1. 仅在需要查询内部文档、公司制度、产品手册等信息时调用
2. 调用时会显示"正在检索知识库..."
3. 检索结果会包含引用来源，请在回答中标注
```

## 9. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 大文件上传超时 | 文档处理失败 | 分片上传 + 后台异步处理 |
| 向量化内存溢出 | 服务崩溃 | 批量处理 + 内存监控 |
| 检索结果不准确 | 用户体验差 | Hybrid Search（语义+关键词） |
