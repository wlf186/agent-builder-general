## 1. 概述

### 1.1 背景

目前 Agent Builder 系统中的智能体仅依赖 Prompt 和外部 API（MCP/Skills），缺乏对企业私有文档的深度理解能力。当用户询问关于公司内部制度、产品手册、技术文档等问题时，智能体无法提供准确答案。

### 1.2 目标

在系统层面建立统一的知识库管理模块，并将 RAG（检索增强生成）抽象为一种可插拔的"工具"，由智能体根据任务需求自主触发检索。

### 1.3 核心价值

- **知识沉淀**: 将企业文档转化为可检索的知识资产
- **精准回答**: 基于实际文档内容生成回答，减少幻觉
- **灵活挂载**: 按需为不同智能体配置不同知识库
- **引用溯源**: 回答可追溯至原始文档来源

## 2. 功能模块

### 2.1 全局知识库管理（系统主页）

#### 2.1.1 知识库 CRUD

| 操作 | 描述 | 必填字段 |
|------|------|----------|
| 创建 | 新建知识库 | 名称、描述（Description） |
| 查看 | 列表展示所有知识库 | - |
| 编辑 | 修改知识库元数据 | 名称、描述 |
| 删除 | 删除知识库及其所有文档 | 需二次确认 |

**描述字段重要性**：描述信息是 LLM 判断是否调用该知识库的核心依据，需引导用户编写清晰、具体的描述。

**示例描述**：
- ✅ "人力资源库：包含公司员工手册、考勤制度、报销流程、年假规定等人力资源相关文档"
- ❌ "HR库"

#### 2.1.2 文档管理

| 操作 | 描述 | 支持格式 |
|------|------|----------|
| 上传 | 上传文档到知识库 | PDF, DOCX |
| 列表 | 查看知识库内所有文档 | - |
| 删除 | 删除单个文档 | - |
| 重建索引 | 重新处理文档向量化 | - |

**文档状态**：
- `pending`: 等待处理
- `processing`: 正在解析/向量化
- `ready`: 可用于检索
- `failed`: 处理失败（显示错误原因）

#### 2.1.3 预处理流水线（后端自动执行）

```
上传文档 → 文本提取 → 分块(Chunking) → 向量化(Embedding) → 存入向量库
```

**分块策略**：
- 默认块大小：512 tokens
- 重叠大小：50 tokens
- 支持按段落/标题智能分块

### 2.2 智能体配置关联（配置页）

#### 2.2.1 知识库挂载

在智能体配置页的"工具/技能"区域新增"RAG 知识库"分类：

```
┌─────────────────────────────────────────────────────────────┐
│  工具与技能配置                                              │
├─────────────────────────────────────────────────────────────┤
│  □ MCP 服务                                                 │
│    ☑ calculator                                             │
│    ☑ cold-jokes                                             │
│                                                             │
│  □ 技能 (Skills)                                            │
│    ☐ AB-PDF Processing Guide                                │
│    ☐ AB-DOCX Processing Guide                               │
│                                                             │
│  ▼ RAG 知识库  [新增分类]                                    │
│    ☐ 人力资源库 - 包含员工手册、考勤制度...                   │
│    ☐ 产品文档库 - 包含产品说明书、API文档...                  │
│    ☐ 技术规范库 - 包含代码规范、架构文档...                   │
└─────────────────────────────────────────────────────────────┘
```

#### 2.2.2 可见性控制

- 只有被勾选的知识库，其描述信息才会进入该智能体的 `Tools Definition`
- 未勾选的知识库对智能体完全不可见，确保数据隔离
- 支持多对多关联：一个知识库可被多个智能体挂载，一个智能体可挂载多个知识库

### 2.3 动态触发逻辑（运行时）

#### 2.3.1 自主决策机制

智能体在 Planning 阶段，根据用户问题和知识库描述判断是否需要检索：

```
用户问: "公司有几天年假？"
    ↓
LLM 分析: 检测到"年假"关键词，匹配"人力资源库"描述
    ↓
触发工具调用: search_knowledge_base(kb_name="人力资源库", query="公司年假规定")
    ↓
检索结果: [文档片段1, 文档片段2, ...]
    ↓
LLM 生成回答: "根据《2026员工手册》第3章第2节，正式员工享有..."
```

#### 2.3.2 工具 Schema 定义

```json
{
  "name": "search_knowledge_base",
  "description": "搜索指定的知识库以获取相关文档内容。仅在与知识库描述相关的查询时使用。",
  "parameters": {
    "type": "object",
    "properties": {
      "kb_name": {
        "type": "string",
        "description": "知识库名称，可选值: [人力资源库, 产品文档库, ...]"
      },
      "query": {
        "type": "string",
        "description": "搜索查询语句"
      }
    },
    "required": ["kb_name", "query"]
  }
}
```

#### 2.3.3 引用溯源

智能体回答时，需在文末或通过脚注标明引用来源：

```
根据《2026员工手册》第3章第2节规定：

> 正式员工入职满一年后，享有5天带薪年假；
> 工作满5年，享有10天带薪年假；
> 工作满10年，享有15天带薪年假。

---
📚 **参考来源**:
- 2026员工手册.pdf (第12-13页)
```

## 4. 验收标准

### 4.1 核心验收场景

| # | 场景 | 预期结果 |
|---|------|----------|
| 1 | 主页创建"人力资源库"知识库 | 知识库创建成功，显示在列表中 |
| 2 | 上传《2026员工手册.pdf》 | 文档处理完成后状态变为"就绪" |
| 3 | 在"行政助手"Agent中勾选该库 | 配置保存成功 |
| 4 | 对话测试："公司有几天年假？" | Agent 显示"正在检索人力资源库..."，并给出基于手册的准确回答 |
| 5 | 在未挂载该库的"技术支持"Agent中询问相同问题 | Agent 回答"不知道"或不触发检索 |
| 6 | 回答中包含引用来源 | 显示"参考来源: 2026员工手册.pdf" |

### 4.2 性能要求

| 指标 | 要求 |
|------|------|
| 文档上传响应 | < 3秒（异步处理） |
| 单文档处理时间 | < 30秒（10页PDF） |
| 检索响应时间 | < 2秒 |
| 检索准确率 | Top-3 召回率 > 80% |

## 5. 非功能性需求

### 5.1 安全性

- 知识库数据隔离：不同知识库的数据完全隔离
- 权限控制：未来版本支持知识库级别的访问控制
- 敏感信息：文档内容不记录到对话历史日志

### 5.2 可扩展性

- 支持后续扩展更多文档格式（Excel, PPT, Markdown）
- 支持后续集成更多向量数据库
- 支持后续添加更多 Embedding 模型

### 5.3 可维护性

- 提供知识库健康检查 API
- 提供索引重建功能
- 提供处理失败日志查询
