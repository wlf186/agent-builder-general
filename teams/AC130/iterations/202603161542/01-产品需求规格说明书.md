## 1. 文档概述

### 1.1 目的
本文档定义"知识库（RAG）管理系统及动态挂载机制"的产品需求，明确功能范围、用户场景和验收标准。

### 1.2 背景
当前 Agent Builder 系统已支持：
- 多种 LLM 模型服务
- MCP 工具集成
- Skill 技能系统
- 对话历史管理
- 文件上传功能

**缺失能力**：
- 无法让智能体基于私有知识库回答问题
- 无法对文档进行语义检索
- 无法动态挂载/卸载知识库

### 1.3 目标
构建一个轻量级 RAG（Retrieval-Augmented Generation）知识库系统，使智能体能够：
1. 接收用户上传的文档（PDF/TXT/MD/DOCX）
2. 自动进行文档分块和向量化
3. 基于用户问题检索相关文档片段
4. 将检索结果注入 LLM 上下文进行回答

## 2. 用户角色与场景

### 2.1 用户角色

| 角色 | 描述 | 核心需求 |
|------|------|----------|
| **知识库管理员** | 企业知识管理者 | 批量上传文档、管理知识库、配置检索参数 |
| **智能体开发者** | 构建智能应用的开发者 | 为智能体挂载知识库、测试检索效果 |
| **终端用户** | 与智能体对话的最终用户 | 获得基于知识库的准确回答 |

### 2.2 核心用户场景

#### 场景 1：企业知识问答
> **用户痛点**：公司规章制度散落在多个文档，员工查找困难

**使用流程**：
1. 管理员创建"公司制度"知识库
2. 上传员工手册、报销制度、考勤规则等 PDF 文档
3. 创建智能体"制度助手"，挂载该知识库
4. 员工提问"年假如何申请？"，智能体基于知识库给出准确答案

#### 场景 2：产品技术文档助手
> **用户痛点**：产品 API 文档频繁更新，客服难以掌握最新信息

**使用流程**：
1. 管理员创建"API 文档"知识库
2. 上传 API 参考文档（Markdown/PDF）
3. 创建智能体"技术支持助手"，挂载该知识库
4. 客服提问时，智能体检索最新的 API 文档并生成回答

#### 场景 3：学习资料问答
> **用户痛点**：学习资料多，难以快速定位知识点

**使用流程**：
1. 用户上传课程讲义、教材摘录到个人知识库
2. 创建学习助手智能体
3. 学习过程中随时提问，系统定位相关知识点

### 3.2 功能详情

#### F1: 知识库管理

| 功能编号 | F1.1 |
|----------|------|
| 功能名称 | 创建知识库 |
| 优先级 | P0（必须） |
| 描述 | 用户可以创建新的知识库，指定名称、描述和存储配置 |

**输入参数**：
- `name`: 知识库名称（唯一标识，2-50字符）
- `description`: 知识库描述
- `embedding_model`: 向量化模型（默认: `bge-small-zh-v1.5`）

**输出**：
- 知识库 ID
- 创建时间

---

| 功能编号 | F1.2 |
|----------|------|
| 功能名称 | 删除知识库 |
| 优先级 | P0（必须） |
| 描述 | 删除知识库及其所有文档数据 |

**前置条件**：
- 无智能体挂载该知识库

---

| 功能编号 | F1.3 |
|----------|------|
| 功能名称 | 知识库列表 |
| 优先级 | P0（必须） |
| 描述 | 获取所有知识库的摘要信息 |

**返回字段**：
- 知识库 ID、名称、描述
- 文档数量、总字符数
- 创建/更新时间

---

#### F2: 文档管理

| 功能编号 | F2.1 |
|----------|------|
| 功能名称 | 上传文档 |
| 优先级 | P0（必须） |
| 描述 | 支持上传单个或批量文档到指定知识库 |

**支持的格式**：
- PDF (.pdf)
- 纯文本 (.txt)
- Markdown (.md)
- Word 文档 (.docx)

**处理流程**：
1. 验证文件格式和大小（单文件 ≤ 10MB）
2. 提取文本内容
3. 文本分块（chunk_size=500, overlap=50）
4. 向量化
5. 存储到向量数据库

**状态反馈**：
- 实时显示处理进度
- 处理完成后显示文档统计（块数、字符数）

---

| 功能编号 | F2.2 |
|----------|------|
| 功能名称 | 删除文档 |
| 优先级 | P1（重要） |
| 描述 | 从知识库中删除指定文档 |

---

| 功能编号 | F2.3 |
|----------|------|
| 功能名称 | 文档列表 |
| 优先级 | P1（重要） |
| 描述 | 获取知识库中的所有文档 |

**返回字段**：
- 文档 ID、文件名、大小
- 上传时间、处理状态
- 块数量

---

#### F3: 向量检索

| 功能编号 | F3.1 |
|----------|------|
| 功能名称 | 语义检索 |
| 优先级 | P0（必须） |
| 描述 | 基于用户问题检索最相关的文档片段 |

**输入参数**：
- `query`: 用户问题
- `top_k`: 返回结果数量（默认 3，范围 1-10）
- `score_threshold`: 相似度阈值（默认 0.6，范围 0-1）

**输出**：
- 匹配的文档片段列表
- 每个片段包含：内容、来源文档、相似度分数

---

| 功能编号 | F3.2 |
|----------|------|
| 功能名称 | 混合检索 |
| 优先级 | P2（可选） |
| 描述 | 结合语义检索和关键词检索（BM25）提高准确率 |

---

#### F4: 动态挂载

| 功能编号 | F4.1 |
|----------|------|
| 功能名称 | 挂载知识库到智能体 |
| 优先级 | P0（必须） |
| 描述 | 将知识库挂载到指定智能体，智能体对话时可检索知识库 |

**配置参数**：
- `knowledge_bases`: 挂载的知识库 ID 列表
- `retrieval_config`: 检索配置
  - `top_k`: 检索结果数量
  - `score_threshold`: 相似度阈值
  - `prompt_template`: 注入模板

---

| 功能编号 | F4.2 |
|----------|------|
| 功能名称 | 卸载知识库 |
| 优先级 | P0（必须） |
| 描述 | 从智能体卸载知识库 |

---

| 功能编号 | F4.3 |
|----------|------|
| 功能名称 | 运行时检索注入 |
| 优先级 | P0（必须） |
| 描述 | 智能体对话时自动检索相关内容并注入上下文 |

**工作流程**：
```
用户问题 → 向量化检索 → Top-K 片段 → 注入 System Prompt → LLM 生成回答
```

**注入模板示例**：
```
请基于以下知识库内容回答用户问题：

<knowledge_base>
{retrieved_chunks}
</knowledge_base>

用户问题：{user_query}
```

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 文档上传响应 | < 100ms | 上传接口返回 |
| 文档处理速度 | > 1 页/秒 | PDF 解析 + 向量化 |
| 检索响应时间 | < 500ms | 包含向量化 + 相似度计算 |
| 并发支持 | 10 个同时上传 | 不影响正常对话 |

### 4.2 存储要求

| 数据类型 | 存储方案 | 说明 |
|----------|----------|------|
| 原始文档 | 文件系统 | `data/knowledge_bases/{id}/documents/` |
| 文档块元数据 | JSON | `data/knowledge_bases/{id}/chunks.json` |
| 向量索引 | ChromaDB | `data/knowledge_bases/{id}/vectordb/` |

### 4.3 兼容性要求

- 支持 Python 3.11+
- 嵌入模型使用本地部署（无需外部 API）
- 向量数据库使用轻量级本地方案（ChromaDB）

### 4.4 安全要求

- 文件上传大小限制（单文件 10MB）
- 文件类型白名单验证
- 知识库隔离（不同智能体只能访问挂载的知识库）

## 6. 约束与风险

### 6.1 技术约束

- 必须使用本地嵌入模型（网络不可用）
- 向量数据库必须轻量级（单机部署）
- 不能增加系统部署复杂度

### 6.2 潜在风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 嵌入模型质量低 | 检索准确率下降 | 提供模型切换选项 |
| 大文件处理慢 | 用户体验差 | 异步处理 + 进度反馈 |
| 向量索引过大 | 磁盘占用高 | 定期清理机制 |
