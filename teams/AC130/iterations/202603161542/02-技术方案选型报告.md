## 1. 概述

### 1.1 技术目标

构建一个轻量级、本地化的 RAG 系统，满足以下要求：
- 无需外部向量数据库服务
- 支持离线部署
- 与现有 Agent Builder 架构无缝集成
- 低资源占用

### 1.2 技术决策维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 部署复杂度 | ⭐⭐⭐⭐⭐ | 优先选择零配置或简单配置方案 |
| 本地化能力 | ⭐⭐⭐⭐⭐ | 必须支持完全离线运行 |
| 性能 | ⭐⭐⭐ | 满足基本使用即可，非核心诉求 |
| 扩展性 | ⭐⭐ | 单租户场景，无需考虑大规模扩展 |
| 社区活跃度 | ⭐⭐⭐ | 影响问题排查和学习成本 |

## 2. 技术选型

### 2.1 向量数据库选型

#### 候选方案对比

| 方案 | 优势 | 劣势 | 评分 |
|------|------|------|------|
| **ChromaDB** | • 轻量级，嵌入式部署<br>• Python 原生，API 简洁<br>• 自带持久化 | • 性能一般（万级数据） | ⭐⭐⭐⭐⭐ |
| **FAISS** | • Meta 出品，性能强<br>• 支持多种索引类型 | • 需要自己管理元数据<br>• 无原生持久化 | ⭐⭐⭐ |
| **pgvector** | • 复用现有 PostgreSQL<br>• 事务支持 | • 需要额外安装 PostgreSQL<br>• 配置复杂 | ⭐⭐ |
| **Milvus** | • 企业级性能<br>• 功能完善 | • 部署复杂（需要 Docker）<br>• 资源占用大 | ⭐⭐ |
| **Qdrant** | • 性能与易用性平衡<br>• 支持 Rust | • 需要独立服务<br>• 增加部署复杂度 | ⭐⭐⭐ |

#### 推荐方案：ChromaDB

**决策理由**：
1. **零配置部署**：`pip install chromadb` 即可使用
2. **嵌入式运行**：无需独立服务，与后端同进程
3. **与 LangChain 深度集成**：已有成熟组件
4. **自动持久化**：数据存储在本地文件系统

**代码示例**：
```python
import chromadb

# 初始化（自动创建 ./chroma_db 目录）
client = chromadb.PersistentClient(path="./data/vectordb")

# 创建集合
collection = client.get_or_create_collection("knowledge_base")

# 添加向量
collection.add(
    documents=["文档内容1", "文档内容2"],
    ids=["doc1", "doc2"],
    embeddings=[[...], [...]]  # 可选，自动生成
)

# 检索
results = collection.query(
    query_texts=["用户问题"],
    n_results=3
)
```

### 2.2 嵌入模型选型

#### 候选方案对比

| 方案 | 优势 | 劣势 | 评分 |
|------|------|------|------|
| **BGE-M3 (bge-m3-v2-zh)** | • 中文效果最佳<br>• 支持多语言<br>• 输入长度 8192 | • 模型较大（约 2GB） | ⭐⭐⭐⭐⭐ |
| **BGE-Small (bge-small-zh-v1.5)** | • 轻量级（约 400MB）<br>• 速度快<br>• 效果良好 | • 最大输入 512 token | ⭐⭐⭐⭐⭐ |
| **text2vec-base-chinese** | • 中文专用<br>• 社区活跃 | • 效果略逊于 BGE | ⭐⭐⭐⭐ |
| **M3E (m3e-base)** | • 中文效果不错<br>• 轻量 | • 维度固定（768） | ⭐⭐⭐ |

#### 推荐方案：BGE-Small-ZH v1.5

**决策理由**：
1. **体积小**：约 400MB，下载快速
2. **速度快**：512 token 限制对文档分块友好
3. **效果好**：在中文语义检索任务上表现优秀
4. **本地运行**：通过 `sentence-transformers` 或 `FlagEmbedding`

**模型规格**：
| 参数 | 值 |
|------|-----|
| 维度 | 512 |
| 最大输入 | 512 tokens |
| 模型大小 | ~400MB |
| 基座 | BERT-base-chinese |

**下载链接**：
```bash
# 自动下载（首次运行时）
HuggingFace: https://huggingface.co/BAAI/bge-small-zh-v1.5

# 手动下载（离线环境）
# 下载文件放到 ~/.cache/huggingface/hub/
```

**代码示例**：
```python
from sentence_transformers import SentenceTransformer

# 加载模型（首次会自动下载）
model = SentenceTransformer('BAAI/bge-small-zh-v1.5')

# 编码文本
embeddings = model.encode([
    "这是一段文档内容",
    "这是用户问题"
])

# 查询向量
query_embedding = model.encode("用户的问题")
```

### 2.4 文本分块策略

#### 分块参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| chunk_size | 500 | 每块约 500 字符（中文） |
| chunk_overlap | 50 | 块之间重叠 50 字符 |
| separator | `\n\n` | 优先按段落分割 |

#### 分块算法选择

推荐使用 **LangChain RecursiveCharacterTextSplitter**：

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=50,
    separators=["\n\n", "\n", "。", "！", "？", "；", "，", " ", ""],
    length_function=len,
)

chunks = splitter.split_text("长文档内容...")
```

**优势**：
1. 优先保持段落完整
2. 按中文标点分割
3. 重叠保证上下文连续性

```python
class Embedder:
    """向量化器"""

    def __init__(self, model_name: str = "BAAI/bge-small-zh-v1.5"):
        from sentence_transformers import SentenceTransformer
        self.model = SentenceTransformer(model_name)
        self.dimension = self.model.get_sentence_embedding_dimension()

    def encode(self, texts: List[str]) -> List[List[float]]:
        """编码文本为向量"""

    def encode_single(self, text: str) -> List[float]:
        """编码单个文本"""
```

```python
class KnowledgeBase(BaseModel):
    """知识库配置"""
    kb_id: str = Field(default_factory=lambda: str(uuid.uuid4())[:8])
```

### 3.4 集成点设计

#### 集成点 1：AgentConfig 扩展

```python
class AgentConfig(BaseModel):
    # ... 现有字段 ...

    # 新增：知识库配置
    knowledge_bases: List[str] = Field(
        default_factory=list,
        description="挂载的知识库 ID 列表"
    )
    retrieval_config: Optional[RetrievalConfig] = Field(
        default=None,
        description="检索配置"
    )
```

#### 集成点 2：AgentEngine 扩展

```python
class AgentEngine:
    def __init__(self, ...):
        # ... 现有初始化 ...
        self.kb_manager = kb_manager  # 新增
        self.embedder = embedder      # 新增

    async def _retrieve_for_query(self, query: str) -> str:
        """为查询检索知识库内容"""
        if not self.config.knowledge_bases:
            return ""

        all_results = []
        for kb_id in self.config.knowledge_bases:
            kb = self.kb_manager.get_kb(kb_id)
            if kb:
                retriever = Retriever(kb.collection)
                results = retriever.search(
                    query,
                    top_k=self.config.retrieval_config.top_k,
                    score_threshold=self.config.retrieval_config.score_threshold
                )
                all_results.extend(results)

        # 格式化为上下文
        return self._format_retrieved_context(all_results)

    def _format_retrieved_context(self, results: List[RetrievalResult]) -> str:
        """格式化检索结果为上下文字符串"""
        if not results:
            return ""

        chunks = []
        for i, r in enumerate(results, 1):
            chunks.append(f"[{i}] {r.content}\n   来源: {r.filename} (相关度: {r.score:.2f})")

        return "\n".join(chunks)
```

## 4. 技术风险评估

### 4.1 已识别风险

| 风险 | 级别 | 缓解措施 |
|------|------|----------|
| 嵌入模型下载失败 | 中 | 提供离线安装包 + 降级方案 |
| 大文件处理超时 | 中 | 异步处理 + 进度反馈 |
| ChromaDB 并发问题 | 低 | 单进程写，多进程读 |
| 向量索引过大 | 低 | 定期清理 + 压缩 |

### 4.2 降级方案

| 主方案 | 降级方案 | 触发条件 |
|--------|----------|----------|
| BGE-Small | random embeddings | 模型加载失败 |
| ChromaDB | 内存字典 | ChromaDB 不可用 |
| 异步处理 | 同步处理 | 队列满时 |
