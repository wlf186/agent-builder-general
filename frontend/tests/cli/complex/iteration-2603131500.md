### TC-MCP-001: 后端 MCP 库依赖检查

## 步骤

1. 调用 MCP 服务列表 API:
   ```bash
   curl http://localhost:20881/api/mcp-services
   ```
2. 记录返回的服务列表
3. 检查 coingecko 服务是否存在于列表中

## 验证
- API 返回 200
- coingecko 服务存在且 `enabled` 为 true

---

### TC-MCP-002: CoinGecko MCP 连接测试 API

## 步骤

1. 调用 CoinGecko 连接测试 API:
   ```bash
   curl -X POST http://localhost:20881/api/mcp-services/coingecko/test
   ```
2. 记录返回结果

## 验证
- API 返回 200
- 响应包含 `success` 字段
- 如果 `success` 为 true: 工具数量 (`tools.length` 或 `tool_count`) 大于 0
- 如果 `success` 为 false: 检查 `mcp_available` 字段判断原因

### TC-MCP-003: CoinGecko MCP 诊断 API

## 步骤

1. 调用 CoinGecko 诊断 API:
   ```bash
   curl -X POST http://localhost:20881/api/mcp-services/coingecko/diagnose
   ```
2. 记录诊断报告 JSON

## 验证
- API 返回 200
- 响应包含以下字段: `service_name` (coingecko)、`overall_status`、`timestamp`、`layers`
- `layers` 是数组且长度大于 0
- 每个层级包含 `status` 字段
- 预期层级: CONFIG、DNS、NETWORK、TLS、MCP
